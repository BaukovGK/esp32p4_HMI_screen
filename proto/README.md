# RO Plant HMI — прототип UI

> Кликабельный HTML-прототип будущего UI HMI-панели для ESP32-P4-NANO 10.1″ 1280×800.
> Отделён от LVGL-кода — служит для согласования дизайна с заказчиком/наладчиком
> до начала имплементации.

## Как открыть

```bash
cd proto
python3 -m http.server 8080      # или любой статический сервер
# Открыть в браузере: http://localhost:8080/
```

Либо просто открыть `proto/index.html` двойным кликом — все ресурсы локальные,
сервер не обязателен.

⚠️ Прототип нарисован под **физический размер дисплея 1280×800** — для удобства
просмотра на десктопе включите масштаб 100% или больше. На дисплее с разрешением
меньше 1320×900 экран будет обрезан.

## Структура

| Файл | Что |
|---|---|
| `index.html` | **Главный экран** — мнемосхема процесса + KPI блок + кнопки управления |
| `washing.html` | **Режим промывки** — фазы цикла, температура, действия |
| `settings.html` | **Настройки и уставки** — давления, дозатор, промывка, таймауты, сеть |
| `debug.html` | **Отладка** — журнал алармов, диагностика контроллера, статус Modbus-устройств |
| `style.css` | Дизайн-токены (цвета, типографика, размеры) + общие компоненты |
| `app.js` | i18n словарь RU/EN, переключение темы, mock-data jitter, confirm modal |

## Что работает в прототипе

| Функция | Где |
|---|---|
| **Навигация** между 4 экранами | tab bar внизу (4 вкладки) |
| **Темы light / dark** | кнопка ☾/☀ в правом верхнем углу |
| **Языки RU / EN** | кнопка RU/EN в правом верхнем углу (texts persisted in localStorage) |
| **Живые значения** | KPI-числа на главном — jitter ±N% раз в секунду |
| **Подтверждения** | кнопки Пуск/Стоп/Промывка/Manual открывают modal «Подтвердите действие» |
| **Часы реального времени** | в status bar |

## Что НЕ работает (прототип не подключён к данным)

- Кнопки реально не отправляют команды (только лог в консоли браузера)
- Параметры — синтетические mock-числа
- Аларм-лог статичный (не обновляется по событиям)
- Темп. промывки — не растёт от нагрева
- Modbus устройства показывают одно состояние

### Типографика

| Уровень | Размер | Где |
|---|---|---|
| `xs` | 12px | Подписи, единицы измерения |
| `sm` | 14px | Описания, секцондари |
| `md` | 16px | Body, кнопки |
| `lg` | 20px | Заголовки секций, KPI значения |
| `xl` | 28px | Заголовки экранов, primary КPI |
| `2xl` | 36px | Hero значения (температура промывки) |
| `3xl` | 48px+ | Аларм-banner (если будет на главном) |

### Геометрия (touch-targets)

- **Минимальная кнопка**: 60×60 px (соответствует ~14 мм при 1280×800 на 10.1″)
- **Большая кнопка** (`btn-lg`): 80 px высоты для важных команд
- **Status bar**: 44 px (стандартная высота для notifications)
- **Tab bar**: 64 px (минимум 11 мм для уверенного попадания)

## Дальнейшие шаги

1. **Согласовать с заказчиком/наладчиком** — пройтись по экранам, собрать
   замечания (что добавить, убрать, переставить).
2. **Заполнить `docs/UI_DESIGN_BRIEF.md`** — там 30+ структурированных
   вопросов которые превратятся в полное ТЗ.
3. После заверки — портировать в LVGL:
   - Цветовая палитра → `ui_theme.{c,h}` (через LVGL styles)
   - Дизайн-токены геометрии → константы в `ui_common.h`
   - Layout каждого экрана → `screens/scr_*.c`
   - i18n → `i18n/lang_ru.c` + `lang_en.c` (тот же словарь, что в `app.js`)
   - Live-data → подписки на `plant_data_t` (через `dirty_flags`)
