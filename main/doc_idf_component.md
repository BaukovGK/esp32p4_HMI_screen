# idf_component.yml

**Путь:** `main/idf_component.yml`

**Назначение:** Манифест зависимостей компонента для менеджера пакетов ESP-IDF Component Manager.

---

## Описание

Файл `idf_component.yml` определяет внешние зависимости проекта, которые автоматически загружаются менеджером компонентов ESP-IDF при сборке. Это аналог `package.json` (npm) или `requirements.txt` (pip) для экосистемы ESP-IDF.

При выполнении `idf.py build` менеджер компонентов автоматически загружает указанные библиотеки из реестра компонентов Espressif (https://components.espressif.com/).

---

## Зависимости

| Компонент | Версия | Источник | Описание |
|-----------|--------|----------|----------|
| `lvgl/lvgl` | ~9.2 | Реестр компонентов | Графическая библиотека LVGL версии 9.2.x |
| `espressif/esp_lvgl_port` | ^2 | Реестр компонентов | Интеграция LVGL с ESP-IDF (порт дисплея, тача, задача LVGL) |
| `espressif/esp_lcd_touch` | ^1.1 | Реестр компонентов | Базовый API для сенсорных контроллеров LCD |
| `idf` | >=5.3.0 | Локальный | Минимальная версия ESP-IDF фреймворка |

---

## Версионирование

Используется семантическое версионирование (SemVer):

| Оператор | Пример | Описание |
|----------|--------|----------|
| `~` | `~9.2` | Совместимые патч-версии: 9.2.0, 9.2.1, ..., но не 9.3.0 |
| `^` | `^2` | Совместимые минорные версии: 2.0.0, 2.1.0, ..., но не 3.0.0 |
| `>=` | `>=5.3.0` | Любая версия начиная с 5.3.0 |

## Подробное описание компонентов

### lvgl/lvgl (~9.2)

Графическая библиотека LVGL (Light and Versatile Graphics Library) версии 9.2.x. Предоставляет:
- Виджеты: кнопки, метки, таблицы, графики, индикаторы
- Систему стилей и тем
- Анимации и переходы между экранами
- Рендеринг шрифтов (включая кириллицу)
- Обработку событий ввода (тач, клавиатура, энкодер)

### espressif/esp_lvgl_port (^2)

Официальный порт LVGL для ESP-IDF. Обеспечивает:
- Создание FreeRTOS-задачи для LVGL tick и отрисовки
- Регистрацию дисплеев (RGB, SPI, I2C, MIPI DSI) в LVGL
- Регистрацию устройств ввода (тач, энкодер) в LVGL
- Потокобезопасный доступ к LVGL через мьютекс

### espressif/esp_lcd_touch (^1.1)

Базовый API для сенсорных контроллеров. Предоставляет:
- Унифицированный интерфейс `esp_lcd_touch_handle_t`
- Чтение координат касания
- Поддержку мультитач
- Трансформацию координат (swap, mirror)

Конкретный драйвер `esp_lcd_touch_gsl3680` подключается через `REQUIRES` в `CMakeLists.txt`.

### idf (>=5.3.0)

Минимальная версия ESP-IDF фреймворка. Версия 5.3.0+ необходима для:
- Полной поддержки ESP32-P4
- API MIPI DSI (`esp_lcd_mipi_dsi.h`)
- Нового I2C master API (`i2c_master.h`)
- Встроенного LDO-регулятора (`esp_ldo_regulator.h`)

## Взаимодействие компонентов

```
idf_component.yml
    |
    +-- lvgl/lvgl ~9.2
    |       |
    |       +-- Отрисовка UI: экраны, виджеты, стили
    |       +-- Обработка тач-ввода
    |
    +-- espressif/esp_lvgl_port ^2
    |       |
    |       +-- display_init.c: lvgl_port_add_disp_dsi()
    |       +-- touch_init.c:   lvgl_port_add_touch()
    |       +-- Задача LVGL tick (FreeRTOS)
    |
    +-- espressif/esp_lcd_touch ^1.1
    |       |
    |       +-- touch_init.c: esp_lcd_touch_config_t
    |       +-- Унифицированный интерфейс для GSL3680
    |
    +-- idf >=5.3.0
            |
            +-- MIPI DSI, I2C master, LDO, Ethernet, MQTT, NVS
```

## Связь с другими файлами

- **CMakeLists.txt** — `REQUIRES` перечисляет компоненты ESP-IDF (встроенные), а `idf_component.yml` — внешние зависимости из реестра
- **display_init.c** — использует `esp_lvgl_port` для регистрации DSI-дисплея в LVGL
- **touch_init.c** — использует `esp_lcd_touch` и `esp_lvgl_port` для регистрации тача
- **ui_main.c** — использует API LVGL 9.2 для создания экранов и виджетов
