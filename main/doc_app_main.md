# app_main.c

**Путь:** `main/app_main.c`

**Назначение:** Точка входа прошивки HMI-дисплея установки обратного осмоса на базе ESP32-P4.

---

## Описание

Файл `app_main.c` содержит единственную функцию `app_main()`, которая является точкой входа FreeRTOS-приложения на ESP32. Функция выполняет последовательную инициализацию всех подсистем HMI-дисплея в строго определённом порядке. После завершения инициализации управление передаётся задачам LVGL (отрисовка UI) и MQTT (обмен данными с основным контроллером).

Файл обёрнут в `#ifndef LVGL_LIVE_PREVIEW` — при сборке в режиме предпросмотра LVGL (без реального железа) этот файл не компилируется.

---

## Последовательность инициализации

| Шаг | Подсистема | Функция | Критичность |
|-----|-----------|---------|-------------|
| 1 | NVS Flash | `nvs_flash_init()` | Критично — без NVS не работают драйверы |
| 2 | Ethernet | `eth_init()` | Некритично — UI работает без сети |
| 3 | Хранилище данных | `plant_data_init()` | Критично — структуры данных для UI |
| 4 | Буфер аварий | `alarm_ring_init()` | Критично — хранение истории аварий |
| 5 | Язык | `lang_init(LANG_RU)` | Критично — русскоязычный интерфейс |
| 6 | Дисплей | `display_init()` | Критично — без дисплея выход из функции |
| 7 | Тач | `touch_init(disp)` | Некритично — можно работать без тача |
| 8 | UI-фреймворк | `ui_init(disp)` | Критично — создание всех экранов |
| 9 | Ожидание IP | `eth_wait_for_ip()` | Некритично — таймаут 10 секунд |
| 10 | MQTT-клиент | `mqtt_client_start()` | Некритично — UI работает без MQTT |

---

## Функции

### `void app_main(void)`

Главная функция приложения (точка входа FreeRTOS). Выполняет последовательную инициализацию всех подсистем. Не принимает параметров, не возвращает значения. При критических ошибках (NVS, дисплей) — завершает работу.

**Обработка ошибок:**
- NVS: при повреждении — автоматическое стирание и повторная инициализация
- Ethernet: при ошибке — предупреждение в лог, продолжение работы
- Дисплей: при ошибке — `return` (работа невозможна)
- Тач: при ошибке — предупреждение, продолжение без сенсорного ввода
- MQTT: при ошибке — лог ошибки, UI продолжает работу

## Зависимости

| Модуль | Заголовок | Назначение |
|--------|-----------|-----------|
| display_init | `display_init.h` | Инициализация MIPI DSI дисплея + LVGL |
| touch_init | `touch_init.h` | Инициализация сенсорного контроллера GSL3680 |
| eth_init | `eth_init.h` | Инициализация Ethernet-соединения |
| plant_data | `plant_data.h` | Хранилище параметров установки (давления, расходы, состояния) |
| alarm_ring | `alarm_ring.h` | Кольцевой буфер аварийных событий |
| lang | `lang.h` | Интернационализация (i18n), выбор русского языка |
| ui_main | `ui_main.h` | Создание и управление LVGL-экранами |
| mqtt_app | `mqtt_app.h` | MQTT-клиент (подписка на `ro_plant/status/#`, `ro_plant/alarms`) |

## Поток данных

```
app_main()
    |
    +-- NVS Flash (хранилище настроек)
    +-- Ethernet --> eth_wait_for_ip() --> MQTT Client
    |                                        |
    +-- plant_data_init() <--- MQTT парсер получает данные --+
    +-- alarm_ring_init() <--- MQTT парсер получает аварии --+
    +-- lang_init(RU)
    +-- display_init() --> lv_display_t *disp
    +-- touch_init(disp) --> lv_indev_t *indev
    +-- ui_init(disp) --> экраны: мнемосхема, параметры, промывки, аварии, настройки, диагностика
```
