# touch_init.c / touch_init.h

**Путь:** `main/board/touch_init.c`, `main/board/touch_init.h`

**Назначение:** Инициализация ёмкостного сенсорного контроллера GSL3680 (I2C) и регистрация в LVGL.

---

## Описание

Модуль выполняет настройку ёмкостного мультитач-контроллера GSL3680, подключённого по шине I2C. Контроллер обеспечивает сенсорный ввод для 10.1-дюймового дисплея HMI-панели установки обратного осмоса.

Шина I2C инициализируется в режиме master с частотой 400 кГц (Fast Mode). Используются встроенные подтяжки (internal pull-up), что позволяет работать без внешних резисторов подтяжки на линиях SDA и SCL.

Координаты сенсорного контроллера зеркалируются по оси X (`mirror_x = 1`) для согласования с программным поворотом дисплея на 270 градусов.

Файл обёрнут в `#ifndef LVGL_LIVE_PREVIEW` -- при предпросмотре тач не инициализируется.

---

## Функции

### `static esp_err_t i2c_init(void)`

Инициализация шины I2C в режиме master.

- Создаёт шину I2C0 с заданными параметрами
- Включает встроенные подтяжки (pull-up) на линиях SDA и SCL
- Настраивает фильтр помех: импульсы короче 7 тактов игнорируются
- При повторном вызове возвращает `ESP_OK` без повторной инициализации (защита через проверку `s_i2c_bus`)

**Конфигурация I2C:**

| Параметр | Значение | Описание |
|----------|----------|----------|
| Порт | I2C_NUM_0 | I2C0 |
| SDA | GPIO_NUM_7 | Линия данных |
| SCL | GPIO_NUM_8 | Линия тактирования |
| Частота | 400 кГц | Fast Mode |
| Фильтр помех | 7 тактов | Игнорирование коротких импульсов |
| Подтяжки | Встроенные | Internal pull-up включены |

**Возвращает:** `ESP_OK` при успехе, код ошибки при неудаче.

---

### `lv_indev_t *touch_init(lv_display_t *disp)`

Полная инициализация тач-контроллера GSL3680 и регистрация в LVGL.

**Последовательность инициализации:**

| Шаг | Действие | Детали |
|-----|---------|--------|
| 1 | Инициализация I2C | Master, 400 кГц, GPIO7/GPIO8 |
| 2 | Создание I2C panel IO | Конфигурация из макроса `ESP_LCD_TOUCH_IO_I2C_GSL3680_CONFIG()` |
| 3 | Настройка тач-контроллера | Разрешение 800x1280, GPIO сброса/прерывания, зеркалирование по X |
| 4 | Создание хэндла GSL3680 | Через `esp_lcd_touch_new_i2c_gsl3680()` |
| 5 | Регистрация в LVGL | Привязка к указанному дисплею через `lvgl_port_add_touch()` |

**Параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `disp` | `lv_display_t*` | Указатель на LVGL-дисплей, к которому привязывается тач |

**Возвращает:** Указатель на `lv_indev_t`, или `NULL` при ошибке.

## Настройка координат

Сенсорный контроллер GSL3680 передаёт координаты в нативной системе панели (800x1280, портрет). Для согласования с программным поворотом дисплея на 270 градусов применяется зеркалирование:

| Флаг | Значение | Описание |
|------|----------|----------|
| `swap_xy` | 0 | Оси X и Y не меняются местами |
| `mirror_x` | 1 | Зеркалирование по X (согласование с поворотом 270) |
| `mirror_y` | 0 | Зеркалирование по Y не применяется |

---

## Аппаратная схема подключения

```
ESP32-P4
    |
    +-- I2C0 Master (400 кГц)
    |       |
    |       +-- GPIO7 (SDA) <---> GSL3680 SDA
    |       +-- GPIO8 (SCL) ----> GSL3680 SCL
    |
    +-- GPIO22 ----> GSL3680 RST (сброс, активный низкий)
    +-- GPIO21 <---- GSL3680 INT (прерывание, активный низкий)
```
