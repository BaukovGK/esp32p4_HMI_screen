# display_init.c / display_init.h

**Путь:** `main/board/display_init.c`, `main/board/display_init.h`

**Назначение:** Инициализация дисплея 10.1" (MIPI DSI, контроллер JD9365) и регистрация в LVGL.

---

## Описание

Модуль выполняет полную настройку дисплейной подсистемы HMI: от включения питания PHY до регистрации дисплея в графической библиотеке LVGL. Дисплей работает в режиме MIPI DSI с 2 линиями данных, нативное разрешение панели 800x1280 (портрет), программный поворот на 270 градусов обеспечивает ландшафтный режим 1280x800.

Подсветка управляется ШИМ-сигналом через контроллер LEDC (GPIO23, 20 кГц, 10-бит разрешение), что обеспечивает плавную регулировку яркости без видимого мерцания.

Файлы обёрнуты в `#ifndef LVGL_LIVE_PREVIEW` — при предпросмотре дисплей не инициализируется.

---

## Функции

### `static esp_err_t backlight_init(void)`

Инициализация ШИМ-подсветки дисплея через LEDC.

- Настраивает таймер LEDC: низкоскоростной режим, 20 кГц, 10-бит разрешение (0..1023)
- Настраивает канал LEDC на GPIO23 с начальной скважностью 0 (подсветка выключена)
- Частота 20 кГц выбрана выше слышимого диапазона для исключения звукового шума

**Возвращает:** `ESP_OK` при успехе, код ошибки при неудаче.

---

### `static esp_err_t backlight_set(int brightness_percent)`

Установка яркости подсветки.

- Принимает яркость в процентах (0..100)
- Значения за пределами диапазона автоматически ограничиваются (clamp)
- Пересчитывает проценты в 10-битное значение скважности: `duty = (1023 * percent) / 100`

**Параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| `brightness_percent` | `int` | Яркость в процентах (0..100) |

**Возвращает:** `ESP_OK` при успехе, код ошибки при неудаче.

---

### `lv_display_t *display_init(void)`

Полная инициализация дисплея и регистрация в LVGL.

**Последовательность инициализации:**

| Шаг | Действие | Детали |
|-----|---------|--------|
| 1 | Инициализация подсветки | LEDC таймер + канал, подсветка выключена |
| 2 | Питание PHY MIPI DSI | Внутренний LDO, канал 3, 2.5 В |
| 3 | Создание шины MIPI DSI | 2 линии данных, макрос `JD9365_PANEL_BUS_DSI_2CH_CONFIG()` |
| 4 | Создание DBI panel I/O | Командный интерфейс для init-последовательности панели |
| 5 | Настройка DPI-тайминга | 800x1280, 60 Гц, RGB565 |
| 6 | Создание панели JD9365 | Сброс, инициализация, включение отображения |
| 7 | Инициализация LVGL-порта | Создание задачи LVGL, мьютекса |
| 8 | Регистрация в LVGL | Буфер в SPIRAM + DMA, одинарная буферизация |
| 9 | Программный поворот | 270 градусов, ландшафт 1280x800 |
| 10 | Включение подсветки | 100% яркости |

**Возвращает:** Указатель на `lv_display_t`, или `NULL` при ошибке.

## Конфигурация буфера LVGL

| Параметр | Значение | Описание |
|----------|----------|----------|
| Размер буфера | 800 * 80 = 64000 пикселей | Определён в `board.h` |
| Двойная буферизация | Нет | Одинарный буфер для экономии памяти |
| Размещение | SPIRAM (PSRAM) | С DMA-доступом |
| Программный поворот | Включён | Необходим для ландшафтного режима |

## Аппаратная архитектура дисплея

```
ESP32-P4
    |
    +-- MIPI DSI Bus (2 линии данных, 1 Гбит/с на линию)
    |       |
    |       +-- DBI (командный канал) --> init-последовательность JD9365
    |       +-- DPI (видеопоток) ------> 800x1280 @ 60Hz, RGB565
    |
    +-- LDO канал 3 (2.5 В) --> Питание PHY MIPI DSI
    |
    +-- GPIO27 --> Аппаратный сброс LCD-панели
    +-- GPIO23 --> LEDC ШИМ --> Подсветка (20 кГц, 10-бит)
```

## Связь с другими модулями

- **board.h** -- все аппаратные константы (GPIO, разрешение, параметры шины)
- **app_main.c** -- вызывает `display_init()` на шаге 6 инициализации
- **touch_init.c** -- тач-контроллер привязывается к дисплею, возвращённому `display_init()`
- **ui_main.c** -- использует `lv_display_t*` для создания экранов
