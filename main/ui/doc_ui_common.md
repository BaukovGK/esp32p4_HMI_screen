# Документация: ui_common

## Файлы

- **Заголовок:** `main/ui/ui_common.h`
- **Реализация:** `main/ui/ui_common.c`

## Назначение

Общие переиспользуемые компоненты UI: панель аварий (alarm bar), навигационная панель (nav bar) и вспомогательные функции форматирования данных. Эти компоненты используются на всех экранах интерфейса.

## Подробное описание

### Панель аварий (Alarm Bar)

Горизонтальная полоса высотой 40 пикселей в верхней части экрана. Содержит 4 элемента, расположенных в flex-строке с выравниванием `SPACE_BETWEEN`:

```
|  Режим (250px)  |  Текст аварии (flex-grow)  |  MQTT (80px)  |  [RU] (50px)  |
```

1. **Режим работы** (250px) -- текст текущего состояния установки с цветовой индикацией:
   - Основной режим: IDLE/AUTO/WASHING/MANUAL/FAULT
   - Подрежим (через " / "): этапы автоматического пуска, этапы промывки
   - Цвет текста соответствует состоянию через `ui_state_color()`

2. **Текст аварии** (flex-grow=1) -- бегущая строка с круговой прокруткой (`LV_LABEL_LONG_SCROLL_CIRCULAR`):
   - Показывает код наиболее критичной активной аварии из кольцевого буфера
   - Цвет зависит от категории аварии через `ui_alarm_cat_color()`
   - При отсутствии аварий -- пустая строка

3. **Статус MQTT** (80px) -- индикатор подключения к брокеру:
   - `"MQTT:OK"` зелёным (COLOR_STATE_AUTO) при подключении
   - `"MQTT:--"` красным (COLOR_STATE_FAULT) при обрыве связи

4. **Кнопка языка** (50x30) -- переключатель RU/EN:
   - При нажатии вызывает `lang_set()` для смены языка интерфейса
   - Обновляет свой текст ("RU" или "EN")

### Навигационная панель (Navigation Bar)

Горизонтальная полоса высотой 60 пикселей в нижней части экрана. Содержит 6 кнопок-вкладок, расположенных в flex-строке с выравниванием `SPACE_EVENLY`:

```
| Мнемосхема | Параметры | Промывка | Аварии | Настройки | Диагностика |
```

Каждая кнопка:
- Размер: 190x50 пикселей
- Поддерживает состояние `LV_STATE_CHECKED` (флаг `LV_OBJ_FLAG_CHECKABLE`)
- Обычный цвет: `COLOR_BG_WIDGET` (тёмно-синий)
- Активный цвет: `COLOR_ACCENT` (голубой)
- Текст -- локализованное название экрана из таблицы `nav_labels[]`

При нажатии кнопки вызывается зарегистрированный callback `nav_cb_t` с индексом экрана.

### Вспомогательные функции форматирования

Функции используются всеми экранами для единообразного форматирования данных:

- `ui_fmt_float()` -- форматирование вещественных значений датчиков с единицей измерения. При NaN (обрыв датчика) выводит `"--- <unit>"`.
- `ui_fmt_uptime()` -- форматирование времени работы в формат `Xd HH:MM:SS` или `HH:MM:SS`.

## Типы

### `alarm_bar_ctx_t` (статическая)

```c
typedef struct {
    lv_obj_t *mode_label;   // Метка режима работы
    lv_obj_t *alarm_label;  // Метка текста аварии
    lv_obj_t *mqtt_label;   // Метка статуса MQTT
    lv_obj_t *lang_btn;     // Кнопка переключения языка
} alarm_bar_ctx_t;
```

### `nav_bar_ctx_t` (статическая)

```c
typedef struct {
    lv_obj_t *btns[6];  // Массив кнопок-вкладок
    nav_cb_t  cb;        // Callback при нажатии
    int       active;    // Индекс активной вкладки
} nav_bar_ctx_t;
```

### `nav_cb_t` (публичная)

```c
typedef void (*nav_cb_t)(int screen_id);
```

Тип callback-функции навигации. Параметр `screen_id` -- индекс выбранного экрана (0..5).

## Функции (публичные)

### `lv_obj_t *ui_alarm_bar_create(lv_obj_t *parent)`

Создание панели аварий. Создаёт flex-контейнер шириной `BOARD_DISP_H_RES` и высотой 40px, выровненный по верхнему левому углу. Заполняет контекст `s_abar` ссылками на дочерние виджеты.

**Параметры:**
- `parent` -- родительский объект (активный экран LVGL)

**Возвращает:** указатель на созданный объект панели.

### `void ui_alarm_bar_update(lv_obj_t *bar, const plant_data_t *data)`

Обновление панели аварий. Параметр `bar` не используется (работа через статический контекст `s_abar`). Должна вызываться внутри `plant_data_lock()/unlock()`.

**Логика обновления:**
1. Режим -- основной текст + подрежим (для AUTO и WASHING)
2. Авария -- код самой критичной из `alarm_ring_get_active()`
3. MQTT -- `"MQTT:OK"` / `"MQTT:--"` с цветовой индикацией

### `lv_obj_t *ui_nav_bar_create(lv_obj_t *parent, nav_cb_t on_select)`

Создание навигационной панели с 6 кнопками. Регистрирует callback для обработки нажатий.

**Параметры:**
- `parent` -- родительский объект
- `on_select` -- callback при выборе вкладки

**Возвращает:** указатель на созданный объект панели.

### `void ui_nav_bar_set_active(lv_obj_t *bar, int screen_id)`

Установка активной вкладки. Снимает `LV_STATE_CHECKED` со всех кнопок, устанавливает на указанную. Параметр `bar` не используется.

### `void ui_fmt_float(char *buf, size_t len, float val, int decimals, const char *unit)`

Форматирование вещественного числа с единицей измерения. При `isnan(val)` выводит `"--- <unit>"` для индикации неисправности датчика.

### `void ui_fmt_uptime(char *buf, size_t len, int64_t seconds)`

Форматирование времени работы. При наличии дней: `"Xd HH:MM:SS"`, иначе: `"HH:MM:SS"`.

## Статические переменные

| Переменная | Тип | Описание |
|---|---|---|
| `s_abar` | `alarm_bar_ctx_t` | Контекст панели аварий (ссылки на виджеты) |
| `s_nav` | `nav_bar_ctx_t` | Контекст навигационной панели (кнопки, callback) |
| `nav_labels[6]` | `str_id_t[]` | Идентификаторы локализованных названий вкладок |

## Интеграция с LVGL

- **Flex layout:** Обе панели используют `LV_FLEX_FLOW_ROW` для горизонтального расположения элементов
- **Бегущая строка:** Метка аварии использует `LV_LABEL_LONG_SCROLL_CIRCULAR` для автоматической прокрутки длинного текста
- **Checkable кнопки:** Навигационные кнопки используют `LV_OBJ_FLAG_CHECKABLE` + `LV_STATE_CHECKED` для индикации активной вкладки
- **user_data:** В навигационных кнопках индекс экрана передаётся через `(void*)(intptr_t)i`
- **flex-grow:** Метка аварии использует `lv_obj_set_flex_grow(1)` для занятия оставшегося пространства
