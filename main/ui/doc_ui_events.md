# Документация: ui_events

## Файлы

- **Заголовок:** `main/ui/ui_events.h`
- **Реализация:** `main/ui/ui_events.c`

## Назначение

Модуль обработчиков событий UI. Содержит callback-функции для кнопок и переключателей на экранах HMI. Все команды управления отправляются на основной контроллер (ESP32-S3) через MQTT-протокол.

## Подробное описание

### Архитектура обработки событий

Модуль разделён на три логические группы:

1. **Диалог подтверждения** -- модальный оверлей с кнопками ДА/НЕТ для опасных операций
2. **Управление режимом** -- callback-функции для переключения режимов установки (AUTO, MANUAL, WASHING, STOP)
3. **Ручное управление** -- переключатели насосов, дозатора и нагревателя

### Диалог подтверждения

Для предотвращения случайного запуска опасных операций (пуск установки, смена режима, промывка) используется модальный диалог подтверждения.

**Механизм работы:**
1. Callback кнопки вызывает `show_confirm_dialog(message_id, mqtt_cmd)`
2. Создаётся полупрозрачный оверлей на `lv_layer_top()` размером 1280x800
3. Оверлей блокирует клики на элементы за ним (`LV_OBJ_FLAG_CLICKABLE`)
4. На оверлее размещается панель диалога 500x250 с заголовком, текстом и кнопками
5. Контекст (`confirm_ctx_t`) хранит MQTT-команду, выделяется через `lv_malloc()`
6. **Кнопка "ДА":** отправляет команду через `mqtt_publish_mode_cmd()`, освобождает ресурсы
7. **Кнопка "НЕТ":** закрывает диалог без действия, освобождает ресурсы

### Управление режимом через MQTT

Команды режима отправляются как строки через `mqtt_publish_mode_cmd()`:

| Callback | MQTT-команда | Диалог | Описание |
|---|---|---|---|
| `ui_evt_start_auto` | `"start_auto"` | Да | Запуск автоматического режима |
| `ui_evt_stop` | `"stop"` | Да | Остановка установки |
| `ui_evt_set_manual` | `"set_manual"` | Да | Переход в ручной режим |
| `ui_evt_start_washing` | `"start_washing"` | Да | Запуск химической промывки |
| `ui_evt_confirm_wash` | `"confirm_wash"` | Нет | Подтверждение этапа промывки |
| `ui_evt_reset_fault` | `"reset_fault"` | Нет | Сброс аварии |

### Ручное управление оборудованием

| Callback | MQTT-функция | Описание |
|---|---|---|
| `ui_evt_pump_toggle` | `mqtt_publish_pump_mask(mask)` | Управление насосами через битовую маску DO |
| `ui_evt_doser_toggle` | `mqtt_publish_doser_enable(bool)` | Включение/отключение дозатора антискаланта |
| `ui_evt_heater_toggle` | `mqtt_publish_heater(bool)` | Включение/отключение нагревателя раствора |

### Управление насосами (ui_evt_pump_toggle)

Особенность: управление насосами реализовано через битовую маску дискретных выходов (DO):

1. `user_data` события содержит индекс бита (0..4), приведённый через `(intptr_t)`
2. Читается текущая маска `do_bits` из `plant_data_t` (под мьютексом, таймаут 10мс)
3. Устанавливается или сбрасывается нужный бит в зависимости от состояния переключателя
4. Новая маска отправляется через `mqtt_publish_pump_mask()`

## Структуры

### `confirm_ctx_t`

```c
typedef struct {
    const char *mqtt_cmd;  // MQTT-команда для отправки при подтверждении
} confirm_ctx_t;
```

Контекст диалога подтверждения. Выделяется через `lv_malloc()`, хранится в `user_data` оверлея, освобождается при закрытии диалога.

## Функции (публичные)

### `void ui_evt_start_auto(lv_event_t *e)`
Callback кнопки "Запуск АВТО". Показывает диалог подтверждения, при подтверждении отправляет `"start_auto"`.

### `void ui_evt_stop(lv_event_t *e)`
Callback кнопки "Стоп". Показывает диалог подтверждения, при подтверждении отправляет `"stop"`.

### `void ui_evt_set_manual(lv_event_t *e)`
Callback кнопки "Ручной режим". Показывает диалог подтверждения, при подтверждении отправляет `"set_manual"`.

### `void ui_evt_start_washing(lv_event_t *e)`
Callback кнопки "Промывка". Показывает диалог подтверждения, при подтверждении отправляет `"start_washing"`.

### `void ui_evt_confirm_wash(lv_event_t *e)`
Callback кнопки "Подтвердить промывку". Отправляет `"confirm_wash"` немедленно (без диалога).

### `void ui_evt_reset_fault(lv_event_t *e)`
Callback кнопки "Сброс аварии". Отправляет `"reset_fault"` немедленно (без диалога).

### `void ui_evt_pump_toggle(lv_event_t *e)`
Callback переключателя насоса. `user_data` содержит индекс бита (0..4). Модифицирует маску DO и отправляет через MQTT.

### `void ui_evt_doser_toggle(lv_event_t *e)`
Callback переключателя дозатора. Отправляет состояние через `mqtt_publish_doser_enable()`.

### `void ui_evt_heater_toggle(lv_event_t *e)`
Callback переключателя нагревателя. Отправляет состояние через `mqtt_publish_heater()`.

## Функции (статические)

### `static void show_confirm_dialog(str_id_t message_id, const char *mqtt_cmd)`
Создание модального диалога подтверждения с полупрозрачным оверлеем.

### `static void confirm_yes_cb(lv_event_t *e)`
Callback кнопки "ДА" -- отправляет MQTT-команду и закрывает диалог.

### `static void confirm_close(lv_event_t *e)`
Callback кнопки "НЕТ" -- закрывает диалог без действия.

## Интеграция с LVGL

- **События:** Все callback-функции подключаются через `lv_obj_add_event_cb()` на событие `LV_EVENT_CLICKED`
- **Слои:** Диалог подтверждения создаётся на `lv_layer_top()`, что гарантирует отображение поверх всех экранов
- **user_data:** Используется для передачи контекста (overlay для диалога, индекс бита для насосов)
- **Состояние:** Переключатели используют `LV_STATE_CHECKED` для определения текущего положения
- **Условная компиляция:** `ESP_LOGI` заменяется на пустой макрос при сборке для LVGL Live Preview
