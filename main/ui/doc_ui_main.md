# Документация: ui_main

## Файлы

- **Заголовок:** `main/ui/ui_main.h`
- **Реализация:** `main/ui/ui_main.c`

## Назначение

Главный модуль пользовательского интерфейса HMI-дисплея установки обратного осмоса. Управляет жизненным циклом экранов (создание, переключение, удаление) и организует поэтапную инициализацию всех компонентов UI.

## Подробное описание

### Архитектура экранов

Модуль реализует паттерн "таблица дескрипторов" для управления экранами. Каждый экран описывается структурой `screen_desc_t`, содержащей два указателя на функции:

- `create(parent)` -- создаёт виджеты экрана внутри родительского контейнера, возвращает корневой объект
- `update(container, data)` -- обновляет виджеты по текущим данным установки `plant_data_t`

Таблица `s_screens[SCREEN_COUNT]` индексируется по `screen_id_t` и содержит дескрипторы всех 6 экранов.

### Компоновка экрана (1280x800)

```
+------------------------------------------+
|  Alarm Bar (40px)                        |  <- s_alarm_bar
+------------------------------------------+
|                                          |
|  Content Area (700px)                    |  <- s_content
|  (текущий экран создаётся здесь)         |     s_screen_obj
|                                          |
+------------------------------------------+
|  Navigation Bar (60px)                   |  <- s_nav_bar
+------------------------------------------+
```

### Поэтапная инициализация

Функция `ui_init()` выполняется в 5 фаз с `vTaskDelay(10ms)` между ними для стабильной работы LVGL на ESP32-P4:

1. **Тема и фон** -- `ui_theme_init()`, тёмный фон `COLOR_BG_DARK`
2. **Панель аварий** -- `ui_alarm_bar_create()` (верхняя полоса 40px)
3. **Контент + навигация** -- контейнер 1280x700 + `ui_nav_bar_create()` (нижние 60px)
4. **Стартовый экран** -- `SCREEN_DIAGNOSTICS` (самый лёгкий для быстрого старта)
5. **Таймер обновления** -- `lv_timer_create(250ms)` для периодического обновления данных

### Переключение экранов

Функция `ui_switch_screen(id)`:
1. Проверяет допустимость id и отличие от текущего экрана
2. Захватывает мьютекс LVGL (`lvgl_port_lock`)
3. Удаляет объект текущего экрана (`lv_obj_delete`)
4. Создаёт новый экран из таблицы дескрипторов
5. Обновляет подсветку активной вкладки навигации
6. Освобождает мьютекс LVGL

### Периодическое обновление

Таймер LVGL `refresh_timer_cb` вызывается каждые 250 мс:
1. Пытается захватить мьютекс `plant_data` с таймаутом 5 мс
2. Обновляет панель аварий (всегда)
3. Обновляет текущий экран через `update()` из дескриптора
4. Освобождает мьютекс

## Перечисление screen_id_t

| Значение | Описание |
|---|---|
| `SCREEN_MNEMONIC` (0) | Мнемосхема технологического процесса |
| `SCREEN_PARAMETERS` (1) | Параметры датчиков (давление, расход, проводимость) |
| `SCREEN_WASHING` (2) | Управление химической промывкой мембран |
| `SCREEN_ALARMS` (3) | Журнал аварий и предупреждений |
| `SCREEN_SETTINGS` (4) | Настройки уставок и параметров |
| `SCREEN_DIAGNOSTICS` (5) | Диагностика системы (память, аптайм, Modbus) |
| `SCREEN_COUNT` (6) | Общее количество экранов |

## Функции

### `void ui_init(lv_display_t *disp)`

Инициализация пользовательского интерфейса. Выполняет 5 фаз создания виджетов с задержками между ними. Параметр `disp` не используется напрямую (дисплей берётся через `lv_display_get_default()`).

### `void ui_switch_screen(screen_id_t id)`

Переключение на указанный экран. Удаляет текущий экран, создаёт новый и обновляет навигацию. Игнорирует повторное переключение на тот же экран и недопустимые id.

## Статические переменные

| Переменная | Тип | Описание |
|---|---|---|
| `s_current` | `screen_id_t` | Идентификатор текущего активного экрана |
| `s_content` | `lv_obj_t*` | Контейнер области контента (700px) |
| `s_alarm_bar` | `lv_obj_t*` | Объект панели аварий |
| `s_nav_bar` | `lv_obj_t*` | Объект навигационной панели |
| `s_screen_obj` | `lv_obj_t*` | Корневой объект текущего экрана |

## Интеграция с LVGL

- **Мьютекс:** Все операции с виджетами LVGL оборачиваются в `lvgl_port_lock(0)` / `lvgl_port_unlock()` для потокобезопасности в среде FreeRTOS
- **Таймер:** Используется `lv_timer_create()` для периодического обновления -- таймер исполняется в контексте задачи LVGL
- **Условная компиляция:** Весь файл обёрнут в `#ifndef LVGL_LIVE_PREVIEW` для поддержки режима предпросмотра в десктопном симуляторе
