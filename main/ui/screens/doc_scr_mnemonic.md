# Экран "Мнемосхема" (scr_mnemonic)

## Файлы
- `main/ui/screens/scr_mnemonic.h` -- заголовок (API)
- `main/ui/screens/scr_mnemonic.c` -- реализация

## Назначение
Главный экран HMI -- P&ID-схема процесса двухступенчатого обратного осмоса.
Отображает в реальном времени состояние всего оборудования, значения датчиков,
телеметрию и предоставляет кнопки управления режимом работы установки.

```
  +-- tank_permeate (frame 80x220) + tank_permeate_fill
  +-- ind_pump1 (circle 60px) -- подающий насос
  +-- ind_pump2 (circle 60px) -- насос 1й ступени
  +-- ind_pump3 (circle 60px) -- насос 2й ступени
  +-- ind_heater (circle 50px) -- нагреватель
  +-- ind_doser (circle 50px) -- дозатор
  +-- membrane M1 (rect 100x80) + membrane M2
  +-- pipe (горизонтальные, 7 сегментов)
  +-- pipe_thin (концентрат/рецикл, 7 сегментов)
  +-- sensor_box P1..P4 (выноски давлений, над трубой)
  +-- sensor_box T (выноска температуры, под трубой)
  +-- sensor_box Q1..Q4 (выноски расходов, под трубой)
  +-- sensor_box S1..S3 (выноски электропроводности)

  |     +-- lbl_recovery (степень извлечения)
  |     +-- lbl_filter_dp (перепад на фильтре)
  +-- lbl_state (текущий режим)
  +-- btn_start, btn_stop, btn_manual, btn_washing, btn_reset
```

## Функции

| Функция | Описание |
|---|---|
| `scr_mnemonic_create(parent)` | Создаёт все виджеты схемы, возвращает корневой контейнер |
| `scr_mnemonic_update(cont, data)` | Обновляет виджеты по данным plant_data_t |
| `make_tank(...)` | Создаёт виджет ёмкости с индикатором заполнения |
| `make_indicator(...)` | Создаёт круглый индикатор оборудования |
| `make_pipe(...)` | Горизонтальный основной трубопровод |
| `make_pipe_v(...)` | Вертикальный основной трубопровод |
| `make_pipe_thin(...)` | Тонкий трубопровод рецикла/концентрата |
| `make_membrane(...)` | Блок мембраны |
| `make_sensor_box(...)` | Выноска датчика с линией к трубе |
| `make_mode_btn(...)` | Кнопка управления режимом |
| `set_indicator_color(...)` | Устанавливает цвет индикатора по DO/DI/fault |
| `set_tank_fill(...)` | Устанавливает высоту заполнения ёмкости |

## Источники данных (plant_data_t)

| Поле | Виджет | Назначение |
|---|---|---|
| `di` (uint8_t) | tank_*_fill, ind_pump* | Дискретные входы (уровни, подтверждения) |
| `do_bits` (uint8_t) | ind_pump*, ind_heater | Дискретные выходы (команды) |
| `pressure[0..3]` | lbl_p[0..3] | Давление P1..P4 (bar) |
| `temperature` | lbl_t | Температура T (C) |
| `flow[0..3]` | lbl_q[0..3] | Расход Q1..Q4 (m3/h) |
| `conductivity[0..2]` | lbl_s[0..2] | Электропроводность S1..S3 (uS/cm) |
| `telemetry.recovery_sys` | lbl_recovery | Степень извлечения (%) |
| `telemetry.filter_dp` | lbl_filter_dp | Перепад на фильтре (bar) |
| `doser.state` | ind_doser | Состояние дозатора |
| `state` | lbl_state, btn_reset | Текущий режим установки |

## Взаимодействие пользователя

| Кнопка | Обработчик | MQTT-команда |
|---|---|---|
| СТАРТ АВТО | `ui_evt_start_auto` | `cmd/start_auto` |
| СТОП | `ui_evt_stop` | `cmd/stop` |
| РУЧНОЙ | `ui_evt_set_manual` | `cmd/manual` |
| ПРОМЫВКА | `ui_evt_start_washing` | `cmd/start_wash` |
| СБРОС АВАРИИ | `ui_evt_reset_fault` | `cmd/reset` |

Кнопка СБРОС АВАРИИ скрыта по умолчанию и показывается только при `state == PLANT_STATE_FAULT`.

## Механизм обновления
- Периодический (таймер ~500 мс в ui_main.c)
- `scr_mnemonic_update()` вызывается из UI-таймера после `plant_data_lock()`
- Не использует событийную модель -- всегда полный перерисовка значений
