# Экран "Аварии" (scr_alarms)

## Файлы
- `main/ui/screens/scr_alarms.h` -- заголовок (API)
- `main/ui/screens/scr_alarms.c` -- реализация

## Назначение
Отображение активных аварий и журнала истории аварий.
Позволяет оператору видеть текущие проблемы и сбросить аварийное состояние.

## Компоновка экрана (1280 x 700 px)

```
+---------------------------------------------------------------------+
| Active Alarms (3)                                          y=8      |
| +---------------------------------------------------------------+   |
| | [|] ! P1_HIGH_PRESS  | 12.50  | 14:32:05 | ACTIVE            |   |
| | [|] ! PUMP2_NO_CONF  | ---    | 14:31:58 | ACTIVE            |   |
| | [|] * FILTER_DP_WARN | 0.45   | 14:30:12 | ACTIVE            |   |
| +---------------------------------------------------------------+   |
|                                                           y=36-276  |
| Alarm History                                             y=290     |
| +---------------------------------------------------------------+   |
| | [|] ! P1_HIGH_PRESS  | 12.50  | 14:32:05 |                   |   |
| | [|] i SYSTEM_START   | ---    | 14:00:00 |                   |   |
| | ...                                                           |   |
| +---------------------------------------------------------------+   |
|                                                           y=318-608 |
| [СБРОС АВАРИИ]                                           y=630     |
+---------------------------------------------------------------------+
```

## Иерархия виджетов

```
cont (1280x700, COLOR_BG_DARK)
  +-- lbl_active_title ("Active Alarms (N)")
  +-- active_list (1240x240, FLEX_COLUMN, прокручиваемый)
  |     +-- alarm_row * N (динамически создаются)
  |           +-- bar (6x28, цвет категории)
  |           +-- icon (символ LVGL)
  |           +-- code (текст кода аварии)
  |           +-- val (числовое значение)
  |           +-- ts (время HH:MM:SS)
  |           +-- act ("ACTIVE" / пусто)
  +-- lbl_history_title ("Alarm History")
  +-- history_list (1240x290, FLEX_COLUMN, прокручиваемый)
  |     +-- alarm_row * M (динамически создаются)
  +-- btn_reset (220x54, скрыта по умолчанию)
```

## Функции

| Функция | Описание |
|---|---|
| `scr_alarms_create(parent)` | Создаёт экран с двумя списками и кнопкой сброса |
| `scr_alarms_update(cont, data)` | Перестраивает оба списка, управляет кнопкой сброса |
| `alarm_cat_color(cat)` | Цвет по категории (CRITICAL/ALARM/WARNING/INFO) |
| `alarm_cat_icon(cat)` | Символ LVGL по категории |
| `format_timestamp(buf, len, ts)` | Unix-время -> "HH:MM:SS" |
| `make_alarm_row(parent, entry)` | Создаёт одну строку аварии |

## Категории аварий

| Категория | Цвет | Иконка |
|---|---|---|
| CRITICAL | COLOR_ALARM_CRITICAL (красный) | WARNING |
| ALARM | COLOR_ALARM_ALARM (оранжевый) | WARNING |
| WARNING | COLOR_ALARM_WARNING (жёлтый) | BELL |
| INFO | COLOR_ALARM_INFO (синий) | EYE_OPEN |

## Источники данных

| Источник | Назначение |
|---|---|
| `alarm_ring_get_active(out, max)` | Массив активных аварий |
| `alarm_ring_get_history(out, max)` | Массив истории аварий |
| `plant_data_t.state` | Видимость кнопки СБРОС (только FAULT) |

## Взаимодействие пользователя

| Кнопка | Видимость | Обработчик | MQTT-команда |
|---|---|---|---|
| СБРОС АВАРИИ | state == FAULT | `ui_evt_reset_fault` | `cmd/reset` |

Оба списка поддерживают вертикальную прокрутку (touch scroll).

## Механизм обновления
- Периодический (таймер ~500 мс)
- При каждом вызове `scr_alarms_update()` оба списка полностью пересоздаются
  (`lv_obj_clean` + создание новых строк). Это допустимо при MAX=16 записей.
