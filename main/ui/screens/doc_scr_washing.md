# Экран "Химическая промывка" (scr_washing)

## Файлы
- `main/ui/screens/scr_washing.h` -- заголовок (API)
- `main/ui/screens/scr_washing.c` -- реализация

## Назначение
Управление 7-фазной химической промывкой мембран обратного осмоса.
Шаговый индикатор показывает прогресс, панель температуры -- текущий/целевой нагрев,
кнопки позволяют запустить, подтвердить фазу ожидания и остановить промывку.

## Компоновка экрана (1280 x 700 px)

```
+---------------------------------------------------------------------+
|                                                                      |
|  (1)----(2)----(3)----(4)----(5)----(6)----(7)    y=80              |
|  Ожид.  Нагрев Ожид.  Подача Ожид.  Дренаж Готово                  |
|  нагр.         подач.        дрен.                                   |
|                                                                      |
|  Текущая фаза: "Нагрев"                           y=240             |
|                                                                      |
|  +--- Температура ---+                             y=300             |
|  | 42.5 / 50.0 C    |  (крупный шрифт)                             |
|  | Target: 50.0 | Max: 60.0 |                                       |
|  +-------------------+                                               |
|                                                                      |
|  [СТАРТ ПРОМЫВКИ] [ПОДТВЕРДИТЬ] [СТОП]            y=536             |
+---------------------------------------------------------------------+
```

## Иерархия виджетов

```
cont (1280x700, COLOR_BG_DARK)
  +-- phase_circle[0..6] (circle 56px) -- круги шагов
  +-- phase_label[0..6] -- подписи фаз под кругами
  +-- phase_connector[0..5] -- линии между кругами
  +-- lbl_phase_name -- текущая фаза (крупный шрифт, y=240)
  +-- temp_panel (500x140, y=300)
  |     +-- temp_title ("Температура")
  |     +-- lbl_temp_current ("42.5 / 50.0 C")
  |     +-- lbl_temp_target ("Target: 50.0 | Max: 60.0")
  +-- btn_start (300x64) -- СТАРТ ПРОМЫВКИ
  +-- btn_confirm (300x64) -- ПОДТВЕРДИТЬ (скрыта по умолчанию)
  +-- btn_stop (300x64) -- СТОП
```

## Функции

| Функция | Описание |
|---|---|
| `scr_washing_create(parent)` | Создаёт экран промывки |
| `scr_washing_update(cont, data)` | Обновляет шаги, температуру, видимость кнопок |
| `wash_sub_to_index(ws)` | Конвертирует enum в 0-based индекс фазы |
| `is_wait_phase(ws)` | Проверяет, нужно ли подтверждение оператора |

## 7 фаз промывки

| Индекс | Enum | Название | Действие |
|---|---|---|---|
| 0 | WASH_SUB_WAIT_HEAT | Ожидание нагрева | Ждёт подтверждения оператора |
| 1 | WASH_SUB_HEATING | Нагрев | Нагреватель работает до целевой T |
| 2 | WASH_SUB_WAIT_SUPPLY | Ожидание подачи | Ждёт подтверждения |
| 3 | WASH_SUB_SUPPLY | Подача раствора | Подача хим. раствора в мембраны |
| 4 | WASH_SUB_WAIT_DRAIN | Ожидание дренажа | Ждёт подтверждения |
| 5 | WASH_SUB_DRAIN | Дренаж | Слив отработанного раствора |
| 6 | WASH_SUB_DONE | Завершено | Промывка окончена |

## Источники данных (plant_data_t)

| Поле | Виджет | Назначение |
|---|---|---|
| `wash_sub` | phase_circle[], phase_label[] | Текущая фаза промывки |
| `temperature.value/.fault` | lbl_temp_current | Текущая температура |
| `set_washing.target_temp_C` | lbl_temp_current/target | Целевая температура |
| `set_washing.max_temp_C` | lbl_temp_current/target | Максимальная температура |
| `state` | btn_start/confirm/stop | Видимость кнопок |

## Взаимодействие пользователя

| Кнопка | Видимость | Обработчик | MQTT-команда |
|---|---|---|---|
| СТАРТ ПРОМЫВКИ | state != WASHING | `ui_evt_start_washing` | `cmd/start_wash` |
| ПОДТВЕРДИТЬ | state == WASHING и фаза WAIT_* | `ui_evt_confirm_wash` | `cmd/confirm_wash` |
| СТОП | state == WASHING | `ui_evt_stop` | `cmd/stop` |

## Цветовая логика шагового индикатора
- Пройдённые фазы: зелёный (COLOR_BTN_SUCCESS)
- Текущая фаза: акцентный (COLOR_ACCENT)
- Будущие фазы: серый (COLOR_BG_WIDGET)
- Соединительные линии: зелёные до текущей, серые после

## Механизм обновления
- Периодический (таймер ~500 мс)
- Полная перерисовка цветов кругов и линий при каждом вызове
