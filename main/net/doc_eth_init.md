# Модуль eth_init -- Инициализация Ethernet

## Расположение файлов

- `main/net/eth_init.c` -- реализация
- `main/net/eth_init.h` -- заголовочный файл (публичный API)

## Назначение

Модуль отвечает за инициализацию проводного Ethernet-соединения на ESP32-P4 с использованием встроенного EMAC-контроллера и внешнего PHY-чипа IP101. Обеспечивает подключение HMI-дисплея к локальной сети для последующей работы MQTT-клиента.

## Подробное описание

### Аппаратная конфигурация

- **MAC**: Встроенный EMAC контроллер ESP32-P4
- **PHY**: IP101 (100 Мбит/с, подключен по SMI/MDIO)
- **GPIO MDC**: задается через `CONFIG_ETH_MDC_GPIO` (Kconfig)
- **GPIO MDIO**: задается через `CONFIG_ETH_MDIO_GPIO` (Kconfig)
- **GPIO Reset PHY**: задается через `CONFIG_ETH_PHY_RST_GPIO` (Kconfig)
- **PHY Address**: задается через `CONFIG_ETH_PHY_ADDR` (Kconfig)

### Последовательность инициализации

1. Создание FreeRTOS EventGroup для сигнализации о получении IP
2. Инициализация TCP/IP стека (`esp_netif_init`)
3. Создание цикла обработки событий (`esp_event_loop_create_default`)
4. Создание сетевого интерфейса Ethernet по умолчанию (`ESP_NETIF_DEFAULT_ETH`)
5. Конфигурация и создание MAC-уровня (внутренний EMAC)
6. Конфигурация и создание PHY-уровня (IP101)
7. Установка Ethernet-драйвера
8. Привязка драйвера к сетевому интерфейсу (netif glue)
9. Регистрация обработчиков событий ETH и IP
10. Запуск Ethernet-драйвера

### Получение IP-адреса

После запуска драйвера, IP-адрес назначается через DHCP (или статическую конфигурацию netif). При получении IP-адреса обработчик события устанавливает бит `ETH_GOT_IP_BIT` в EventGroup, что позволяет другим задачам синхронизироваться через вызов `eth_wait_for_ip()`.

## Функции

### `esp_err_t eth_init(void)`

Полная инициализация Ethernet-подсистемы. Вызывается один раз при старте приложения (из `app_main`).

- **Возвращает**: `ESP_OK` при успехе, код ошибки ESP-IDF при неудаче
- **Побочные эффекты**: Создает EventGroup, инициализирует netif, запускает драйвер

### `esp_err_t eth_wait_for_ip(TickType_t timeout_ticks)`

Блокирующее ожидание получения IP-адреса. Используется для задержки запуска MQTT-клиента до установления сетевого соединения.

- **Параметры**: `timeout_ticks` -- максимальное время ожидания в тиках FreeRTOS
- **Возвращает**: `ESP_OK` если IP получен, `ESP_ERR_TIMEOUT` при таймауте
- **Примечание**: Бит не сбрасывается после чтения -- повторные вызовы возвращают результат мгновенно

### `static void eth_event_handler(...)`

Внутренний обработчик событий Ethernet и IP. Логирует состояние линка и устанавливает бит `ETH_GOT_IP_BIT` при получении IP.

## Макросы и константы

| Макрос | Значение | Описание |
|--------|----------|----------|
| `ETH_GOT_IP_BIT` | `BIT0` | Бит EventGroup, сигнализирующий о получении IP-адреса |

## Конфигурация (Kconfig)

| Параметр | Описание |
|----------|----------|
| `CONFIG_ETH_MDC_GPIO` | Номер GPIO для тактового сигнала SMI (MDC) |
| `CONFIG_ETH_MDIO_GPIO` | Номер GPIO для данных SMI (MDIO) |
| `CONFIG_ETH_PHY_ADDR` | Адрес PHY-чипа на шине SMI |
| `CONFIG_ETH_PHY_RST_GPIO` | Номер GPIO для аппаратного сброса PHY |

## Зависимости

- **ESP-IDF**: `esp_eth`, `esp_event`, `esp_netif`, `esp_log`, `esp_check`, `driver/gpio`
- **FreeRTOS**: EventGroup для синхронизации
- **Аппаратные**: PHY IP101, встроенный EMAC ESP32-P4

## Связь с другими модулями

- Вызывается из `app_main.c` при старте приложения
- После успешной инициализации и получения IP запускается модуль `mqtt_app`

## Обработка ошибок

- Все критические вызовы обернуты в `ESP_RETURN_ON_ERROR` -- при ошибке инициализация прерывается и возвращается код ошибки
- Потеря линка логируется как предупреждение (`ESP_LOGW`)
- Таймаут ожидания IP возвращает `ESP_ERR_TIMEOUT`

## Условная компиляция

Весь файл `eth_init.c` обернут в `#ifndef LVGL_LIVE_PREVIEW` -- при сборке для предпросмотра UI в десктопном эмуляторе LVGL модуль Ethernet исключается из компиляции.
