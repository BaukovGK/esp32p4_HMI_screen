## Контекст железа (фиксировано, для справки)

- Дисплей: **10.1″ IPS 1280×800** (ландшафт после rotation 270°)
- Тач: GT911 ёмкостный, **режим polling** (один палец, без жестов)
- MCU: ESP32-P4 dual-core 400 MHz, 32 MB PSRAM, LVGL 9.2
- Связь: Ethernet → MQTT с контроллером ESP32-S3
- Целевая среда: промышленная RO-установка, **на стенде/в техническом помещении**

## После заполнения

Когда заполните — скажите, я через `design:write-spec` или вручную:
1. Соберу полное **ТЗ** с описанием каждого экрана (компоненты, layout, состояния)
2. Сделаю **wireframe-описания** в виде ASCII-mockup'ов или текстовых блоков
3. Сформирую **дизайн-токены** (цвета, типографика, размеры) для LVGL
4. Создам **матрицу алармов** (приоритет → визуальный паттерн → звук → действие)
5. Опишу **навигационную карту** (state diagram переходов между экранами)
6. Дам **тех-задание** для разработчика LVGL: какие компоненты создать,
   какие события обработать, какие topic'и подписаться

Дополнительно через design-skills:
- `design:accessibility-review` — аудит цветовой схемы под промышленные условия
- `design:ux-copy` — русские строки UI с правильным тоном
- `design:design-system` — формализация дизайн-токенов и компонентов
