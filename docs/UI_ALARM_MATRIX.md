# Матрица алармов — поведение HMI

> Все коды алармов из `docs/api/asyncapi.yaml::AlarmCode` (~30 кодов).
> Эта таблица описывает что делает HMI на каждый аларм:
> визуальный паттерн, звук, действие оператора, требуемый доступ.

---

## 1. Категории и базовое поведение

| Категория | Toast | Banner color | Pulse | Звук | Modal | Доступ к silence |
|---|---|---|---|---|---|---|
| **CRITICAL** (0x0001-0x000F, 0x00B0-0x00B1) | ✅ | red, blink | ✅ KPI | beep × 1, повтор 30с | ✅ | technolog (PIN) |
| **ALARM** (0x0010-0x004F, 0x00C1-0x00C5) | ✅ | red, solid | ✅ KPI | beep × 1, повтор 60с | ✅ | technolog (PIN) |
| **WARNING** (0x0050-0x005F, 0x00C6, 0x00D0-0x00D7) | ✅ | yellow | ❌ | beep × 1 (один раз) | ❌ | technolog (PIN) |
| **INFO** (0x0001-0x000F service) | ❌ (только в журнал) | — | ❌ | ❌ | ❌ | — |

Все категории попадают в **«Алармы»** экран и в журнал событий.

### Pulse-анимация

Применяется к **KPI** (kpi-card на главной/параметрах) когда значение источника аларма выходит за допустимый диапазон. Цикл 1.5 секунды, опасить может только cooled-down silence или клиа.

### Звук

ES8311 кодек на плате, через `i2s_chan_handle_t`:
- **CRITICAL beep:** 1 кГц, 200 мс, повторяется каждые 30 секунд пока активен (отключается командой silence)
- **ALARM beep:** 800 Гц, 200 мс, каждые 60 секунд
- **WARNING beep:** 600 Гц, 100 мс, **один раз** при появлении (без повторов)
- **INFO:** беззвучно

Громкость регулируется через настройки (default 80%). Можно полностью отключить звук в settings.

## 3. Шаблоны toast-сообщений

Формат: `[Категория] {Краткое описание} [{значение если применимо}]`

**Примеры (RU):**
- `АВАРИЯ — E-STOP активен`
- `АВАРИЯ — Высокое давление P3 (38.1 bar)`
- `АВАРИЯ — Перегрев ВД-насоса (84.2 °C)`
- `Внимание — Засорение фильтра (ΔP 1.2 bar)`
- `Внимание — KWS-306L slave 21 offline`
- `Внимание — Мало свободной памяти`

**Примеры (EN):**
- `FAULT — E-STOP active`
- `FAULT — High pressure P3 (38.1 bar)`
- `FAULT — HP pump overheat (84.2 °C)`
- `Warning — Filter clogged (ΔP 1.2 bar)`
- `Warning — KWS-306L slave 21 offline`
- `Warning — Low free memory`

**Длина toast:** не более 60 символов в одну строку (с учётом цифр в скобках).

## 5. Действия с алармами

### 5.1 Заглушение (silence)

**Доступ:** только технолог (PIN).
**Команда:** `ro_plant/command/silence` (любой payload).
**Эффект:**
- Звук прекращается.
- KPI-pulse останавливается.
- Toast убирается из status bar.
- Сам аларм остаётся **active=true** в журнале и в alarm_ring (отображается там).
- При новом аларме (другом коде) — звук восстанавливается.

⚠️ Заглушение — **не сброс**. Аларм остаётся активным до фактического устранения причины.

### 5.2 Сброс истории (clear)

**Доступ:** только технолог (PIN).
**Команда:** `ro_plant/command/alarm_clear` (новая, ⚠️ TODO API).
**Эффект:**
- Очистка `alarm_ring` на стороне HMI.
- Очистка истории на стороне контроллера (опционально, если API поддерживает).
- **Не влияет** на активные алармы — они появятся снова при следующем publish от контроллера.

### 5.3 Авто-сброс (clear от контроллера)

Когда контроллер публикует тот же код с `active=false`:
- Аларм помечается «cleared» в HMI alarm_ring.
- KPI возвращается в норму (зелёный).
- В списке алармов запись остаётся (страйк-через текст или opacity 50%).

## 6. Поведение в разных состояниях UI

| State HMI | Алярм CRITICAL | Алярм ALARM | Алярм WARNING |
|---|---|---|---|
| **Главная (active)** | Toast + Banner + Modal + Pulse + Beep | то же | Toast + Beep×1 |
| **Промывка (active)** | Toast + Banner + Modal + Beep + Auto pause cycle | Toast + Modal + Beep | Toast + Beep×1 |
| **Параметры (active)** | Toast + KPI Pulse + Modal | то же | Toast + KPI цвет |
| **Алармы (active)** | Banner + новая запись в топе списка | то же | новая запись |
| **Settings (active, в форме)** | Toast + Modal сразу (поверх настроек) | то же | Toast |
| **PIN-locked screen** | Toast + Modal + Beep (Modal без действий, просто инфо) | то же | Toast + Beep |
| **Connection lost (offline)** | Старые алармы остаются, новые не приходят | то же | то же |

## 7. Открытые вопросы / TODO API

- **`ro_plant/command/alarm_clear`** — нужен новый MQTT-канал на контроллере для сброса истории алармов (Q4.3 ответ). Добавить в `asyncapi.yaml::channels` и реализовать в `mqtt_subscribe.c`.
- **`ro_plant/hmi/audit`** — новый канал для логирования действий оператора с HMI (Q7.4). Контроллер только пишет в свой лог/InfluxDB, не реагирует.
- **«Резкое изменение» детектор** (Q4.1#2): нужен на стороне контроллера — он рассчитывает derivative (dp/dt, dq/dt, dσ/dt) и публикует новый аларм при превышении порога. Это **не имплементировать в HMI**, только обработать как обычный аларм когда контроллер начнёт его слать.
