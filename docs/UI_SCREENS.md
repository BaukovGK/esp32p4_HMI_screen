# Wireframes: детальное описание экранов

> Каждый экран расписан как: layout, состояния, MQTT-подписки/публикации,
> особенности взаимодействия. Прототип в `proto/` — кликабельная демонстрация
> базовой компоновки.

Все экраны 1280×800 (landscape), общая структура:
- Status bar 44 px (top)
- Content 692 px
- Tab bar 64 px (bottom)

### 1.2 Контент мнемосхемы (SVG/canvas)

По вертикальному потоку из `ro_plant_system_spec_v2.md` §2.1.

**Приборы (датчики)** изображаются по стандарту **ISA-5.1 P&ID** —
круглые индикаторы с двумя строками:
- Верхняя: **тег прибора** (P1, Q3, σ2 — обозначение по типу величины)
- Нижняя: **текущее значение** (число с точностью 1-2 знака)

**Каждый прибор связан с трубопроводом:**
- **leader-line** — тонкая (1 px) линия от круга прибора до точки врезки
- **tap-point** — маленький filled circle (r=3.5 px) на трубопроводе там,
  где формально расположен датчик
- При смене состояния (warn/danger/offline) leader-line, tap-point и
  рамка круга меняют цвет согласованно

Размер круга на мнемосхеме: радиус 22 px (диаметр 44 px), ISA-стандарт
≈11-13 мм для крупного 10″ display. Линия-разделитель посередине круга
(между тегом и значением).

**Стрелки направления потока** на каждом активном сегменте трубы — через
SVG `marker-end` (определены в `<defs>`):
- `arrow-active` — accent-цвет, на активных потоках
- `arrow-recycle` — info-цвет, на рециркуляции / концентрате (dashed)
- `arrow-pipe` — серый, на неактивных трубах (например дренаж)

**Локализованные рециклы** (вместо обхода всей мнемосхемы):
- Рецикл 1-й ст. — петля СВЕРХУ над насосом 1ст и мембраной 1
  (от выхода концентрата мембраны 1 → up → влево → down ко входу насоса 1ст)
- Рецикл 2-й ст. — петля СНИЗУ под насосом 2ст и мембраной 2
  (от концентрата мембраны 2 → down → вправо → up ко входу насоса 2ст)

**Цветовое кодирование рамки/разделителя:**
- зелёный (accent) — норма
- жёлтый (warn) — приближение к порогу
- красный (danger, pulse) — выход за порог + анимация мерцания
- серый dashed (offline) — нет данных или sensor fault

| Объект | Приборы (sensor circles) |
|---|---|
| Исходная ёмкость | DI1 как точка-индикатор (●/○) на ёмкости |
| Преднагнетания (RO1) | Pump-block с badge статуса + 2 круга у выхода: P1, Q1 |
| Дозатор (RO5) | Block с badge OFF/ON, без приборов |
| Фильтр | Block без приборов в самом фильтре, **P2** между фильтром и насосом 1ст |
| Насос 1-й ст. (RO2) | Pump-block + badge + **P3** между насосом и мембраной 1 |
| Мембрана 1 | Block, **σ2** у выхода пермеата вниз, **Q2** на рецикле концентрата |
| Промежуточная | Tank с DI2/DI3 как ●/○ |
| Насос 2-й ст. (RO3) | Pump-block + badge + **P4** между насосом и мембраной 2 |
| Мембрана 2 | Block, **σ3** + **Q3** у выхода пермеата |
| Чистая | Tank с DI4 |
| Рецикл 1ст | Dashed-линия + **Q2** на пути |
| Дренаж 1ст | Dashed → стрелка вниз |
| Рецикл 2ст | Dashed + **Q4** на пути (без дренажа) |

**Итого 8 приборных кругов на мнемосхеме:**
P1, Q1, P2, P3, σ2, Q2, P4, σ3, Q3, Q4 = 10 точек.
σ1 (feed) и σ4 (concentrate) — НЕ на мнемосхеме, только на «Параметры»
(σ1 контекстный для сравнения с σ3, σ4 служебный).

### 1.3 KPI справа (8 ключевых параметров)

Конфигурация **в коде** (по умолчанию), оператор не меняет.
Из 11 источников данных (4P + 4Q + 3σ + T + 2W) на главную выводятся
только критичные для быстрого взгляда:

| KPI | Источник | Норма | Цвет |
|---|---|---|---|
| P3 (1-я ст.) | analog/P3 | < 32 bar | green / yellow @ 32-35 / red @ >35 |
| P4 (2-я ст.) | analog/P4 | < 7 bar | green / red @ >8 |
| σ пермеат тов. | conductivity/s3 | < 5 µS | green / yellow @ 5-10 / red @ >10 |
| σ питат. вода | conductivity/s1 | информ. | grey (контекст для оценки очистки) |
| Q пермеат | flow/Q3 | 1.0-2.0 m³/h | green |
| Темп. | analog/T | 15-30°C | green / yellow @ <15, >30 |
| НД мощность | power/lp/power | 500-1500 W | green |
| ВД мощность | power/hp/power | 1500-3500 W | green |

**Не на главной** (доступны на «Параметры»):
- P1, P2 (промежуточные давления)
- Q1, Q2, Q4 (расходы)
- σ2 (пермеат 1ст — вспомогательный)
- σ4 (концентрат — служебный, может быть скрыт через settings)

### 1.4 Кнопки управления

Дизайн отражает топ-3 сценария Q1.3 (авто запуск / ручной запуск /
сохранение установок). На главной — две **большие primary** кнопки + три
secondary:

| Размер | Кнопка | Команда | Доступ | Confirm? |
|---|---|---|---|---|
| **LARGE** (80×) | ▶ **Пуск AUTO** | `mode = start_auto` | guest | yes |
| **LARGE** (80×) | ⚙ **Ручной режим** | `mode = set_manual` + переход | technolog (PIN) | yes |
| medium (60×) | ⏹ Стоп | `mode = stop` | guest | **no — защитное действие, выполняется сразу** |
| medium (60×) | Промывка | переход на экран Промывка | technolog | no (confirm на самом экране) |
| medium (60×) | 🔇 Заглушить (виден только если есть актив. алармы) | `silence` | technolog | no |

**E-STOP** реализован отдельно в status bar (всегда виден на любом
экране) — это критичный quick-action.

### 1.5 MQTT подписки

```
ro_plant/status/state
ro_plant/status/io
ro_plant/status/analog/P1..P4, T
ro_plant/status/flow/Q1..Q4
ro_plant/status/conductivity/s1..s4
ro_plant/status/power/lp, hp
ro_plant/availability
ro_plant/alarms (для toast и счётчика на вкладке)
```

### 1.6 Состояния

- **Connected + AUTO/RUNNING:** все потоки live, KPI обновляются, badge зелёные.
- **Connected + IDLE:** трубы серые, насосы badge=OFF, кнопка Пуск AUTO подсвечена.
- **Connected + FAULT:** мнемосхема приглушена, status bar красный, доступна только Стоп.
- **Connected + WASHING:** активные потоки только в участке промывки, остальные серые.
- **Disconnected (release):** splash «Нет связи» поверх схемы, кнопки disabled.
- **Disconnected (debug):** UI как обычно, индикатор ●Хаб красный.

### 2.2 Workflow фаз

```
[1] Ожидание ──► [2] Нагрев ──► [3] Подача ──► [4] Слив ──► (back to IDLE)
     │                │                │                │
     │                │                │                │
   user click      auto when         user click       user click
   "Начать"        t°≥35°C           "Подтв.подачу"   "Подтв.слив"
                                     (после ручных    (после ручного
                                      переключений    открытия дренажа)
                                      вентилей)
```

### 2.3 Состояния каждой фазы

- **inactive** (другая активна) — серый, бледный
- **active** — синий бордер, насыщенные цвета, текущий timer
- **completed** — зелёная галочка
- **aborted** — красный X (если оператор отменил)

### 2.4 Действия (по фазе)

| Фаза | Кнопка | Команда | Доступ |
|---|---|---|---|
| Ожидание | «Начать нагрев» | `mode = start_washing` | technolog |
| Нагрев (active) | «Отменить» | `washing/cancel` | technolog |
| Нагрев (✓ ready) | «Подтвердить подачу» | `washing/confirm_circ` | technolog |
| Подача (active) | «Отменить» | `washing/cancel` | technolog |
| Подача (✓ ready) | «Подтвердить слив» | `washing/confirm_drain` | technolog |
| Слив (active) | «Завершить» | `washing/complete` или `mode = stop` | technolog |

- **🗑 Очистить историю** (сверху справа) — PIN-protected, double-confirm, шлёт `alarm_clear` (TODO API).
