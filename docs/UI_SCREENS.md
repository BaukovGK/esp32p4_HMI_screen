> особенности взаимодействия. Прототип в `proto/` — кликабельная демонстрация
> базовой компоновки.

Все экраны 1280×800 (landscape), общая структура:
- Status bar 44 px (top)
- Content 692 px

### 1.2 Контент мнемосхемы (SVG/canvas)

По вертикальному потоку из `ro_plant_system_spec_v2.md` §2.1.

**Приборы (датчики)** изображаются по стандарту **ISA-5.1 P&ID** —
круглые индикаторы с двумя строками:
- Верхняя: **тег прибора** (P1, Q3, σ2 — обозначение по типу величины)
- Нижняя: **текущее значение** (число с точностью 1-2 знака)

**Каждый прибор связан с трубопроводом:**
- **leader-line** — тонкая (1 px) линия от круга прибора до точки врезки
- **tap-point** — маленький filled circle (r=3.5 px) на трубопроводе там,
  где формально расположен датчик
- При смене состояния (warn/danger/offline) leader-line, tap-point и
  рамка круга меняют цвет согласованно

Размер круга на мнемосхеме: радиус 22 px (диаметр 44 px), ISA-стандарт
≈11-13 мм для крупного 10″ display. Линия-разделитель посередине круга
(между тегом и значением).

**Стрелки направления потока** на каждом активном сегменте трубы — через
SVG `marker-end` (определены в `<defs>`):
- `arrow-active` — accent-цвет, на активных потоках
- `arrow-recycle` — info-цвет, на рециркуляции / концентрате (dashed)
- `arrow-pipe` — серый, на неактивных трубах (например дренаж)

**Локализованные рециклы** (вместо обхода всей мнемосхемы):
- Рецикл 1-й ст. — петля СВЕРХУ над насосом 1ст и мембраной 1
  (от выхода концентрата мембраны 1 → up → влево → down ко входу насоса 1ст)
- Рецикл 2-й ст. — петля СНИЗУ под насосом 2ст и мембраной 2
  (от концентрата мембраны 2 → down → вправо → up ко входу насоса 2ст)

**Цветовое кодирование рамки/разделителя:**
- зелёный (accent) — норма
- жёлтый (warn) — приближение к порогу
- красный (danger, pulse) — выход за порог + анимация мерцания
- серый dashed (offline) — нет данных или sensor fault

| Объект | Приборы (sensor circles) |
|---|---|
| Исходная ёмкость | DI1 как точка-индикатор (●/○) на ёмкости |
| Преднагнетания (RO1) | Pump-block с badge статуса + 2 круга у выхода: P1, Q1 |
| Дозатор (RO5) | Block с badge OFF/ON, без приборов |

| Насос 1-й ст. (RO2) | Pump-block + badge + **P3** между насосом и мембраной 1 |
| Мембрана 1 | Block, **σ2** у выхода пермеата вниз, **Q2** на рецикле концентрата |
| Промежуточная | Tank с DI2/DI3 как ●/○ |
| Насос 2-й ст. (RO3) | Pump-block + badge + **P4** между насосом и мембраной 2 |
| Мембрана 2 | Block, **σ3** + **Q3** у выхода пермеата |
| Чистая | Tank с DI4 |
| Рецикл 1ст | Dashed-линия + **Q2** на пути |
| Дренаж 1ст | Dashed → стрелка вниз |
| Рецикл 2ст | Dashed + **Q4** на пути (без дренажа) |

**Итого 10 приборных кругов на мнемосхеме:**
P1, P2, P3, P4 (давления), Q1, Q2, Q3, Q4 (расходы), σ2, σ3 (проводимости).

доступен только через «Параметры».

### 1.2.2 Особенности реализации

**Стандарт ISA-5.1:** все приборы изображены как круги (диаметр 44 px).
Подключение к трубопроводу через leader-line + tap-point (filled circle 3.5 px).
См. подробно `UI_SPEC.md` §9.

**Насосы — centrifugal pump symbol (◯):** каждый насос это
`<g class="pump-circle [state]">` радиуса 26 px с фоном `pump-bg`,
группой `pump-rotor` (4 wedge-blades + центральный hub) и подписью
снизу. Состояния:
- `.running` — `fill: url(#pumpGradRun)` (light→dark green radial),
  ротор крутится 1.5 s
- `.starting` — жёлтый, ротор крутится 3 s + бордер пульсирует 1.2 s
- `.error` — красный, бордер пульсирует 1 s, ротор статичный
- `.off` — серый, статичный

Анимации отключаются по `prefers-reduced-motion: reduce`.
Полная спецификация компонента — `UI_SPEC.md` §9.5.

**Фильтр — ромб с "Ф" (◇):** `<polygon class="filter-diamond">`
40×40 на center (330, 210) с вершинами (330,190)(350,210)(330,230)
(310,210), внутри текст «Ф» 14 px. В половину прежнего блока — освобождает
место для P1/Q1 над трубой между Преднагн и фильтром. Подпись «Фильтр»
ниже на y=248.

**Ёмкости — расширенная графика:** каждая ёмкость это `<g class="tank-group">`
содержит:
1. `<rect class="tank">` — корпус (rounded rect, ~80 высоты)
2. clipped группа: `<rect class="tank-water-grad">` с заливкой
   `url(#tankWaterGrad)` (светло-голубой → синий по вертикали) +
   `<path class="tank-wave-line">` — белая волна 5 s ease-in-out
3. `<text class="tank-name">` — название сверху в воздушной части
4. `<text class="tank-pct">` — % заполнения (с обводкой по `paint-order`)
5. `<line class="tank-probe">` — пунктирный стержень-зонд
6. `<g class="level-sw [.active|.alarm]">` per switch — DI на стержне:
   - default (сухой) — hollow circle серого
   - `.active` (мокрый, контакт замкнут) — filled green
   - `.alarm` — filled red + pulse

Switches: Исходная — DI1 (low, **active**); Промежуточная —
DI3 (high, dry) + DI2 (low, **active**); Чистая — DI4 (high, dry).

**Анимация потока:** на активных трубах — flow-march (бегущие капли,
полупрозрачный dashed-overlay поверх pipe-active). На рециклах —
recycle-march (анимация stroke-dashoffset на dashed-линии).
Стрелки направления НЕ используются.

**Локализованные рециклы:** компактные петли вокруг своих насосов:
- **Рецикл 1ст** — петля Y=100 над row 1, выходит из top-edge мембраны 1
  (650, 180), приходит в junction (420, 210) на pipe-active (фильтр→1ст).
- **Рецикл 2ст** — петля Y=455 под row 2, выходит из **bottom-center
  мембраны 2** (280, 380) **прямо в pipe-recycle** (без pipe-active stub
  и без junction-маркера на источнике — симметрично с рециклом 1).
  Приходит в junction (550, 350) на pipe-active (промеж→2ст).
  Класс `pipe-recycle flowing` без перехода mid-pipe.

**Inject point дозатора:** труба от дозатора (216, 285) идёт горизонтально
до x=435, затем вертикально вверх к junction (435, 210) **на pipe фильтр→1ст**
(а не до фильтра — антискалант не задерживается мехочисткой).

**Outlet-труба дренажа 1ст:** серая `pipe` 80 px от right-edge мембраны 1
(710, 210) → (790, 210), label «дренаж». На 2-й ступени дренажа нет.

### 1.3 KPI справа (8 ключевых параметров)

Конфигурация **в коде** (по умолчанию), оператор не меняет.
Из 11 источников данных (4P + 4Q + 3σ + T + 2W) на главную выводятся
только критичные для быстрого взгляда:

| KPI | Источник | Норма | Цвет |
|---|---|---|---|
| P3 (1-я ст.) | analog/P3 | < 32 bar | green / yellow @ 32-35 / red @ >35 |
| P4 (2-я ст.) | analog/P4 | < 7 bar | green / red @ >8 |
| σ пермеат тов. | conductivity/s3 | < 5 µS | green / yellow @ 5-10 / red @ >10 |
| σ питат. вода | conductivity/s1 | информ. | grey (контекст для оценки очистки) |
| Q пермеат | flow/Q3 | 1.0-2.0 m³/h | green |
| Темп. | analog/T | 15-30°C | green / yellow @ <15, >30 |
| НД мощность | power/lp/power | 500-1500 W | green |
| ВД мощность | power/hp/power | 1500-3500 W | green |

**Не на главной** (доступны на «Параметры»):
- P1, P2 (промежуточные давления)
- Q1, Q2, Q4 (расходы)
- σ2 (пермеат 1ст — вспомогательный)
- σ4 (концентрат — служебный, может быть скрыт через settings)

### 1.4 Кнопки управления

Дизайн отражает топ-3 сценария Q1.3 (авто запуск / ручной запуск /
сохранение установок). На главной — две **большие primary** кнопки + три
secondary:

| Размер | Кнопка | Команда | Доступ | Confirm? |
|---|---|---|---|---|
| **LARGE** (80×) | ▶ **Пуск AUTO** | `mode = start_auto` | guest | yes |
| medium (60×) | ⚙ Ручной режим | `mode = set_manual` + переход | technolog (PIN) | yes |
| medium (60×) | ⏹ Стоп | `mode = stop` | guest | **no — защитное действие, выполняется сразу** |
| medium (60×) | Промывка | прямой переход на `washing.html` (data-href) | technolog | **no** (confirm на самом экране промывки) |
| medium (60×) | 🔇 Заглушить (виден только если есть актив. алармы) | `silence` | technolog | no |

⚠️ Только **«Пуск AUTO»** оставлен как `btn-lg` (LARGE primary).
«Ручной режим» — обычной высоты `--touch-min` (60 px), как «Стоп».
Это снижает визуальный шум и оставляет один яркий call-to-action.

**Промывка** — кнопка использует `data-href="washing.html"` и общий
обработчик `[data-href]` в `app.js`, который делает
`window.location.href = ...` без confirm-modal.

**E-STOP** реализован отдельно в status bar (всегда виден на любом
экране) — это критичный quick-action.

### 1.5.2 Клик на оборудование — модал по фильтру/насосу/мембране

Помимо приборов кликабелен **сам корпус оборудования** — фильтр, все
3 насоса, обе мембраны. Каждый такой блок обёрнут в
`<g class="equipment-clickable" data-equipment="{id}">` и при клике
открывает свой тип детального модала (используется тот же
`#sensor-modal`, переписывается `innerHTML`).

**Идентификаторы и типы:**

| `data-equipment` | Тип модала | Что показывает |
|---|---|---|
| `filter` | filter | ΔP (= P1 − P2) с бейджем + пороги (warn 0.5 bar, alarm 0.8 bar); наработка картриджа (часов / max) с прогресс-баром, оставшееся время до замены, дата установки |
| `pump-pre` | pump | Состояние (RUNNING/STARTING/ERROR/STOPPED), время в текущем цикле, дата запуска, общая наработка (моточасы), число пусков, модель, Q ном, P ном |
| `pump-st1` | pump | (то же) |
| `pump-st2` | pump | (то же) |
| `membrane-1` | membrane | Селективность по NaCl %; наработка от последней промывки (часы / порог) + прогресс + дата + остаток до плановой; общая наработка (часы / ресурс) + прогресс + остаток; тип |
| `membrane-2` | membrane | (то же) |

**CSS:** `.equipment-clickable` ставит `cursor: pointer`; на hover —
`filter: brightness(1.08) drop-shadow(0 0 4px var(--accent))`; на
focus-visible — `stroke: var(--accent-hover)` + `stroke-width: 3`.

**JS:** в `proto/app.js` — словарь `equipmentMeta` с псевдо-данными
для прототипа, три рендерера (`renderFilterModal`, `renderPumpModal`,
`renderMembraneModal`) и диспетчер `showEquipmentDetail(id)`.
Делегирование клика в `DOMContentLoaded` с защитой:

```js
document.querySelectorAll('.mnemo-svg [data-equipment]').forEach(el => {
    el.addEventListener('click', e => {
        // если клик прилетел из вложенного sensor-group — пропускаем
        if (e.target.closest('.sensor-group')) return;
        showEquipmentDetail(el.getAttribute('data-equipment'));
    });
});
```

Хелперы: `formatHours(h)` → "1284 ч (53 дн)";
`renderProgressBar(pct, state)` → инлайн-бар с цветом по state
(`ok`/`warn`/`danger`); `classifyByThresholds(value, warn, alarm)`.

На целевой LVGL-реализации `equipmentMeta` будет заполняться из
контроллера через MQTT (топики `ro_plant/status/<id>/runtime`,

### 1.6 Состояния

- **Connected + AUTO/RUNNING:** все потоки live, KPI обновляются, badge зелёные.
- **Connected + IDLE:** трубы серые, насосы badge=OFF, кнопка Пуск AUTO подсвечена.
- **Connected + FAULT:** мнемосхема приглушена, status bar красный, доступна только Стоп.
- **Connected + WASHING:** активные потоки только в участке промывки, остальные серые.
- **Disconnected (release):** splash «Нет связи» поверх схемы, кнопки disabled.
- **Disconnected (debug):** UI как обычно, индикатор ●Хаб красный.
