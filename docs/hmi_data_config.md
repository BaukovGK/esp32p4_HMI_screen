# Конфигурация данных HMI-дисплея — Контроллер установки обратного осмоса

**Версия:** 1.0
**Дата:** 2026-02-09
**HMI платформа:** ESP32-P4 с дисплеем 10.1"
**Протокол связи:** MQTT через Ethernet switch

---

## 1. Архитектура взаимодействия

```
ESP32-S3 (Контроллер)              ESP32-P4 (HMI)
┌──────────────────────┐           ┌──────────────────────────┐
│  ProcessTask (100мс) │           │  UI Task                 │
│  ├── state_machine   │           │  ├── Экран мнемосхемы    │
│  ├── interlocks      │           │  ├── Экран параметров    │
│  ├── doser           │           │  ├── Экран промывки      │
│  └── telemetry       │           │  ├── Экран аварий        │
│                      │           │  ├── Экран настроек      │
│  MqttTask            │  ◄═MQTT═► │  └── Экран диагностики   │
│  ├── publish (5с)    │           │                          │
│  └── subscribe       │           │  MQTT Client             │
│                      │           │  ├── subscribe status/*  │
│  WebServer           │  ◄═HTTP═► │  └── publish command/*   │
│  └── REST API        │           │                          │
└──────────────────────┘           └──────────────────────────┘
         │                                    │
         └────────── Ethernet Switch ─────────┘
                         │
                   Mosquitto (MQTT)
                   Orange Pi Docker
```

HMI может получать данные двумя способами:
1. **MQTT** (рекомендуется) — подписка на `ro_plant/status/#` и `ro_plant/alarms`
2. **HTTP REST API** — polling GET-запросами к `http://<controller-ip>/api/v1/status`

## 2. Данные для отображения

### 2.1 Состояние системы

**MQTT топик:** `ro_plant/status/state`
**REST API:** `GET /api/v1/status` → поля `state`, `auto_sub`, `wash_sub`, `fault_flags`

| Поле | Тип | Описание | Значения |
|------|-----|----------|----------|
| `state` | string | Текущий режим | `IDLE`, `AUTO`, `WASHING`, `MANUAL`, `FAULT` |
| `auto_sub` | string\|null | Подсостояние AUTO | `STARTING_PUMP1`, `RAMP`, `STARTING_PUMP2`, `FILLING_INTERM`, `STARTING_PUMP3`, `RUNNING`, `STOPPING` |
| `wash_sub` | string\|null | Подсостояние WASHING | `WAIT_HEAT`, `HEATING`, `WAIT_SUPPLY`, `SUPPLY`, `WAIT_DRAIN`, `DRAIN`, `DONE` |
| `fault_flags` | uint32 | Битовая маска аварий | См. раздел 2.8 |

**Рекомендации для UI:**
- Крупный индикатор режима (цвет: IDLE=серый, AUTO=зелёный, WASHING=синий, MANUAL=жёлтый, FAULT=красный)
- При `AUTO` показывать текущий подэтап пуска
- При `WASHING` показывать текущую фазу и кнопку подтверждения (см. раздел 3.3)

### 2.2 Дискретные входы/выходы

**MQTT топик:** `ro_plant/status/io`
**REST API:** `GET /api/v1/status` → поля `di`, `do`

#### Дискретные входы (DI) — 8 бит

| Бит | DI | Назначение | Тип | Нормальное состояние |
|-----|----|------------|-----|---------------------|
| 0 | DI1 | Уровень исходной ёмкости | NC | 1 = есть вода |
| 1 | DI2 | Уровень промежуточной — низкий | NC | 1 = есть вода |
| 2 | DI3 | Уровень промежуточной — высокий | NO | 0 = не полная |
| 3 | DI4 | Уровень пермеатной — высокий | NO | 0 = не полная |
| 4 | DI5 | Аварийный стоп | NC | 1 = норма |
| 5 | DI6 | Подтверждение насоса подачи | NO | 1 = контактор вкл |
| 6 | DI7 | Подтверждение насоса 1-й ступени | NO | 1 = контактор вкл |
| 7 | DI8 | Подтверждение насоса 2-й ступени | NO | 1 = контактор вкл |

#### Дискретные выходы (DO) — 8 бит

| Бит | DO | Назначение | Тип нагрузки |
|-----|----|------------|--------------|
| 0 | RO1 | Насос подачи (преднагнетания) | УПП |
| 1 | RO2 | Насос 1-й ступени (ВД) | УПП |
| 2 | RO3 | Насос 2-й ступени (ВД) | УПП |
| 3 | RO4 | Нагреватель (ТЭН) | Реле |
| 4 | RO5 | Насос-дозатор | Сигнальный |
| 5-7 | RO6-RO8 | Резерв | — |

**Рекомендации для UI:**
- На мнемосхеме: анимация (насос вращается, клапан открыт) при `DO bit=1`
- DI5=0 (E-STOP) — красный мигающий индикатор
- Разрыв между DO и DI подтверждения (DO1=1 но DI6=0) — индикатор ожидания

### 2.3 Аналоговые входы (давления, температура)

**MQTT топики:** `ro_plant/status/analog/P1..P4`, `ro_plant/status/analog/T`
**REST API:** `GET /api/v1/status` → массив `analog[]`

| Канал | Параметр | Единицы | Диапазон | Уставка аварии |
|-------|----------|---------|----------|----------------|
| P1 | Давление после насоса подачи | бар | 0 — 6 | p1_max (5.5) |
| P2 | Давление после фильтра | бар | 0 — 6 | — (расчётный) |
| P3 | Давление после насоса 1-й ст. | бар | 0 — 40 | p3_max (35.0) |
| P4 | Давление после насоса 2-й ст. | бар | 0 — 10 | p4_max (8.0) |
| T | Температура воды | °C | 0 — 100 | t_overshoot (45.0) |

Поля каждого канала:
| Поле | Тип | Описание |
|------|-----|----------|
| `value` | float\|null | Значение в инж. единицах (`null` при NaN) |
| `unit` | string | Единицы измерения |
| `fault` | bool | `true` = неисправность 4–20 мА: обрыв (< 3.5 мА) или короткое замыкание (> 20.5 мА) |

**Рекомендации для UI:**
- Шкала (gauge) или числовое поле с единицами
- При `fault=true` — перечеркнутый значок датчика, текст "ОБРЫВ"
- Цветовая индикация: нормально=зелёный, >80% уставки=жёлтый, >уставки=красный

### 2.5a Счётчики электроэнергии KWS-306L (добавлено 2026-05-09)

| Поле | Тип | Единицы | Описание |

### 2.6 Телеметрия (расчётные параметры)

|-----|-----------|------------------|-------------------|

| `PUMP1_TIMEOUT` | ALARM | Нет подтв. насоса подачи | Проверить контактор/УПП |
| `PUMP2_TIMEOUT` | ALARM | Нет подтв. насоса 1-й ст. | Проверить контактор/УПП |
| `PUMP3_TIMEOUT` | ALARM | Нет подтв. насоса 2-й ст. | Проверить контактор/УПП |
| `SENSOR_FAULT_*` | ALARM | Обрыв датчика P1/P2/P3/P4/T | Проверить проводку датчика |
| `MANUAL_DEP` | WARNING | Нарушение зависимости | Включить предшествующий агрегат |
| `MQTT_DISCONNECT` | INFO | MQTT отключён | Информационное |

**Рекомендации для UI:**
- Панель активных аварий сверху/справа экрана (всегда видна)

- INFO — серый текст (только в журнале)

## 3. Команды управления (HMI → Контроллер)

### 3.1 Команды режима

**MQTT:** Publish в `ro_plant/command/mode`
**REST API:** `POST /api/v1/command` с телом `{"cmd": "..."}`

| Команда | MQTT payload | REST body | Условие |
|---------|-------------|-----------|---------|
| Запуск AUTO | `start_auto` | `{"cmd":"start_auto"}` | Из IDLE |
| Остановка | `stop` | `{"cmd":"stop"}` | Из любого режима |
| Запуск промывки | `start_washing` | `{"cmd":"start_washing"}` | Из IDLE |
| Подтверждение фазы | `confirm_wash` | `{"cmd":"confirm_wash"}` | В WASHING (WAIT_*) |
| Ручной режим | `set_manual` | `{"cmd":"set_manual"}` | Из IDLE |
| Сброс аварии | `reset_fault` | `{"cmd":"reset_fault"}` | Из FAULT (E-STOP снят) |

**Рекомендации для UI:**
- Кнопки доступны только в соответствующих состояниях (остальные серые/скрыты)
- Кнопка `stop` доступна ВСЕГДА (кроме IDLE и FAULT)
- `confirm_wash` — крупная кнопка на экране промывки (видна только при `wash_sub` = `WAIT_*`)

---

### 3.2 Управление в MANUAL

**MQTT:** Publish в `ro_plant/command/pump`
**REST API:** `POST /api/v1/manual/do`

```json
{"mask": 5}
```

`mask` — битовая маска DO (8 бит, бит 0 = RO1). Каждая отправка полностью заменяет текущее состояние DO.

**Пример:** Включить насос подачи (RO1) + ТЭН (RO4):
```json
{"mask": 9}
```
(0b00001001 = бит 0 + бит 3)

| Бит | DO | Элемент UI |
|-----|----|------------|
| 0 | RO1 | Тумблер "Насос подачи" |
| 1 | RO2 | Тумблер "Насос 1-й ступени" |
| 2 | RO3 | Тумблер "Насос 2-й ступени" |
| 3 | RO4 | Тумблер "Нагреватель" |
| 4 | RO5 | Тумблер "Дозатор" |

**Предупреждения зависимости (MANUAL_DEP_WARN):**
| Условие | Предупреждение |
|---------|---------------|
| RO2 вкл, RO1 выкл | Насос 1-й ступени без подачи! |
| RO3 вкл, RO2 выкл | Насос 2-й ступени без 1-й ступени! |
| RO4 вкл, RO1 выкл | Нагреватель без циркуляции! |

---

### 3.3 Управление промывкой (WASHING)

Промывка — 7-фазный процесс с подтверждением оператора перед каждой активной фазой.

```
WAIT_HEAT ──[confirm]──► HEATING ──[T>=target]──► WAIT_SUPPLY ──[confirm]──►
SUPPLY ──[timer]──► WAIT_DRAIN ──[confirm]──► DRAIN ──[timer]──► DONE ──► IDLE
```

#### Экран промывки (рекомендация)

```
┌───────────────────────────────────────────────────────────────┐
│                    ХИМИЧЕСКАЯ ПРОМЫВКА                         │
│                                                               │
│  Текущая фаза: [██ НАГРЕВ ██]  Подача   Слив                │
│                                                               │
│  Температура:  25.0°C  ──────────────── 35.0°C (цель)       │
│  ████████████░░░░░░░░░░░░░░░░░░░░░░░░  71%                  │
│                                                               │
│  ТЭН: ВКЛ      Насос подачи: ВКЛ      Таймаут: 28:45        │
│                                                               │
│               ┌──────────────────────┐                        │
│               │  ПОДТВЕРДИТЬ ПОДАЧУ  │  (появляется в WAIT_*) │
│               └──────────────────────┘                        │
│                                                               │
│  [ОСТАНОВИТЬ ПРОМЫВКУ]                                        │
└───────────────────────────────────────────────────────────────┘
```

#### Действия оператора по фазам

| Фаза | Статус | Действие оператора | Кнопка confirm |
|------|--------|-------------------|----------------|
| `WAIT_HEAT` | Ожидание | Подготовить клапаны для нагрева | Да → HEATING |
| `HEATING` | Нагрев | Контроль температуры | Нет (автоматически) |
| `WAIT_SUPPLY` | Ожидание | Переключить клапаны на подачу через мембраны | Да → SUPPLY |
| `SUPPLY` | Подача (20 мин) | Наблюдение | Нет (по таймеру) |
| `WAIT_DRAIN` | Ожидание | Открыть дренажный клапан | Да → DRAIN |
| `DRAIN` | Дренаж (5 мин) | Наблюдение | Нет (по таймеру) |
| `DONE` | Завершено | — | → автоматически IDLE |

#### Параметры промывки (настраиваемые)

| Параметр | Поле JSON | По умолч. | Диапазон |
|----------|-----------|-----------|----------|
| Целевая температура | `target_temp_C` | 35.0°C | 20 — 40 |
| Макс. температура | `max_temp_C` | 40.0°C | 25 — 45 |
| Аварийная температура | `t_overshoot_C` | 45.0°C | 30 — 60 |
| Гистерезис ТЭНа | `hysteresis_C` | 2.0°C | 0.5 — 10 |
| Таймаут нагрева | `heat_timeout_min` | 30 мин | 5 — 120 |
| Время подачи | `supply_time_min` | 20 мин | 5 — 120 |
| Время дренажа | `drain_time_min` | 5 мин | 1 — 60 |

**Логика гистерезиса ТЭНа:**
- ВКЛ при T < (target - hysteresis)
- ВЫКЛ при T >= target
- В зоне (target-hysteresis)...target — сохраняет текущее состояние

### 3.4 Управление дозатором

Дозатор работает только в `AUTO` режиме по таймеру (N мин работы / M мин цикла).

## 5. REST API (альтернатива MQTT)

Базовый URL: `http://<controller-ip>/api/v1/`

### GET endpoints

| Endpoint | Описание |
|----------|----------|
| `GET /api/v1/status` | Полный статус (state, IO, analog, flow, cond, telemetry, doser, interlocks) |
| `GET /api/v1/config` | Текущая конфигурация (все секции) |
| `GET /api/v1/alarms` | Активные аварии + журнал (последние 16) |
| `GET /api/v1/diagnostics` | Диагностика (heap, stack, modbus, uptime) |
| `GET /api/v1/mqtt/status` | Статус MQTT подключения |

### POST endpoints

| Endpoint | Body | Описание |
|----------|------|----------|
| `POST /api/v1/command` | `{"cmd":"start_auto"}` | Команда режима |
| `POST /api/v1/manual/do` | `{"mask": 5}` | Управление DO (MANUAL) |
| `POST /api/v1/doser/enable` | `{"enabled": true}` | Вкл/выкл дозатора |
| `POST /api/v1/config/pressure` | `{...}` | Настройки давления |
| `POST /api/v1/config/doser` | `{...}` | Настройки дозатора |
| `POST /api/v1/config/washing` | `{...}` | Настройки промывки |
| `POST /api/v1/config/timeouts` | `{...}` | Настройки таймаутов |
| `POST /api/v1/config/mqtt` | `{...}` | Настройки MQTT |

Ответ на все POST: `{"ok": true}` или `{"ok": false, "error": "..."}`.

## 6. Рекомендуемая структура экранов HMI

### Экран 1: Мнемосхема (главный)
- Технологическая схема установки с анимацией
- Значения P1-P4, T, Q1-Q4, σ1-σ4 у соответствующих точек
- Индикаторы уровней ёмкостей (DI1-DI4)
- Статус насосов и ТЭНа (DO + DI подтверждения)
- Панель аварий (справа или сверху)
- Кнопки: START AUTO / STOP / MANUAL / WASHING

### Экран 2: Параметры
- Таблица текущих значений всех аналоговых входов
- Расходомеры: текущий расход + накопленный объём
- Кондуктометры: УЭП + температура
- Расчётные: recovery, selectivity, filter_dp

### Экран 3: Промывка
- Индикатор текущей фазы (прогресс-бар 7 шагов)
- Температура + график (если есть ресурсы)
- Таймер обратного отсчёта для SUPPLY/DRAIN
- Кнопка подтверждения (в фазах WAIT_*)

### Экран 4: Аварии
- Список активных аварий (категория, код, значение, время)
- Журнал: скроллируемый список последних 16 событий
- Кнопка "Сброс аварии" (при FAULT)

### Экран 5: Настройки
- Формы ввода по секциям (pressure, doser, washing, timeouts, mqtt)
- Кнопка "Применить" для каждой секции

### Экран 6: Диагностика
- Free heap / min heap
- Uptime
- Стек задач
- Статус Modbus устройств (online/errors)
- Статус MQTT подключения
